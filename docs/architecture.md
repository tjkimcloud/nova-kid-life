# Architecture — NovaKidLife

## System Overview

NovaKidLife is a serverless, event-driven platform with a statically exported frontend
served via CDN, a REST API via Lambda, and an async content pipeline for scraping,
AI enrichment, image generation, and social posting.

---

## Request Flow (Frontend)

```
User (browser/mobile)
    │
    │  HTTPS  novakidlife.com
    ▼
AWS CloudFront (CDN)
    │   Cache hit → serve immediately
    │   Cache miss → fetch from origin
    ▼
AWS S3  (novakidlife-web)
    │   Static HTML/CSS/JS/fonts
    │   Generated by: next build (static export)
    └── Served as: pre-rendered pages (no SSR)
```

---

## Request Flow (API)

```
Client (browser / mobile / server)
    │
    │  HTTPS  api.novakidlife.com
    ▼
AWS API Gateway  (REST)
    │   Routes: /events, /events/{slug}, /events/search,
    │           /categories, /locations, /health, etc.
    │   Auth: API key (admin routes), public (read routes)
    ▼
AWS Lambda  novakidlife-api  (Python 3.12)
    │
    ├──► Supabase PostgreSQL
    │       events, categories, locations tables
    │       pgvector: embeddings column for semantic search
    │
    └──► SSM Parameter Store  (secrets)
```

## Content Pipeline (Automated, Daily)

```
AWS EventBridge  (cron: 0 6 * * ? *)   ← 6am EST daily
    │
    ▼
Lambda: novakidlife-scheduler  (Python 3.12)
    │   Orchestrates the daily pipeline
    │   Reads config from SSM
    │
    ▼
Lambda: novakidlife-events-scraper  (Python 3.12)
    │   Scrapes N sources (Fairfax Parks, Loudoun, Arlington, Eventbrite...)
    │   Deduplicates via source_url unique constraint
    │   Writes raw events → Supabase (status: raw)
    │
    │   For each new event → publishes message to SQS
    ▼
AWS SQS  novakidlife-events-queue
    │   Visibility timeout: 5 min
    │   DLQ: novakidlife-events-dlq  (after 3 failures)
    │
    ├──────────────────────────────────────────────┐
    ▼                                              ▼
Lambda: novakidlife-image-gen             Lambda: novakidlife-api (enrichment path)
    │   (Python 3.12)                         (or separate enrichment Lambda)
    │   1. Build image prompt                     │
    │   2. Imagen 3  ──► success                  │   gpt-4o-mini
    │      └──► DALL-E 3 fallback                 │   Generates:
    │   3. Upload webp → S3 (novakidlife-media)    │   - title, description
    │   4. Update events.image_url in Supabase     │   - SEO fields
    │                                              │   - tags, age_range
    │                                              │   Updates event in Supabase
    └──────────────────────────────────────────────┘
                            │
                            │  (event fully enriched → status: published)
                            ▼
             AWS SQS  novakidlife-social-queue
                            │
                            ▼
             Lambda: novakidlife-social-poster  (Python 3.12)
                            │   Builds post copy
                            │   Schedules optimal posting time
                            ▼
                    Buffer API
                            │
                  ┌─────────┼─────────┐
                  ▼         ▼         ▼
              Facebook  Instagram  Twitter/X
```

## Data Store

```
Supabase (PostgreSQL 15 + pgvector)
    │
    ├── events              Main events table
    │     id, slug, title, description, short_description
    │     start_at, end_at, location, lat, lng
    │     image_url, source_url, status
    │     age_range, tags[], seo_title, seo_description
    │     embedding vector(1536)    ← pgvector for semantic search
    │     created_at, updated_at
    │
    ├── categories          Event category taxonomy
    │
    ├── locations           NoVa location reference (county, city, coords)
    │
    └── newsletter_subs     Email subscription list

AWS S3  novakidlife-media    (CloudFront: media.novakidlife.com)
    └── events/{id}/hero.webp    AI-generated event images
```

## Security Boundaries

```
Public (no auth):
    GET /events, GET /events/{slug}, POST /events/search
    GET /categories, GET /locations

API-key protected (admin):
    POST /admin/events/trigger-scrape
    GET /admin/health/detailed

Lambda ↔ Supabase:
    Service key from SSM (never in code or env files)

Lambda ↔ AWS services:
    IAM roles with least-privilege (no wildcards in prod)
```
